Welcome to HivePro Documentation

HivePro is a comprehensive security platform designed to protect your applications and infrastructure. This documentation will help you get started and make the most of HivePro's powerful features.

Getting Started

Whether you're new to HivePro or an experienced user, this guide will help you understand the core concepts and get up and running quickly.

Note: This documentation is for HivePro v5.0 and above. For older versions, please refer to the legacy documentation.

Quick Installation

Install HivePro with a single command:

# Using Docker
docker pull hivepro/platform:latest

# Using Kubernetes
kubectl apply -f https://hivepro.io/deploy/k8s.yaml

# Using Package Manager
curl -sSL https://get.hivepro.io | sh

Key Features

HivePro includes powerful features designed for modern security operations:

  • Comprehensive Security: Full-stack security scanning and monitoring
  • Real-time Threat Detection: Advanced detection and response capabilities
  • Compliance Management: Built-in compliance frameworks and reporting
  • Integration Ready: Rich API and webhook support for automation
  • Scalable: Designed for enterprise-scale deployments

Basic Usage

Once installed, you can access HivePro through the web interface or CLI. Here are some basic commands:

# Start a security scan
hivepro scan --target myapp.example.com

# View scan results
hivepro results --latest

# Generate compliance report
hivepro report --framework pci-dss

# List all vulnerabilities
hivepro vulns --severity high
Important: Make sure to configure your scan targets and authentication before running security scans. See the Configuration section for more details.

Configuration

HivePro can be configured through the web UI or by editing the configuration file directly. The config file is located at:

  • Docker: /etc/hivepro/config.yaml
  • Kubernetes: ConfigMap in hivepro namespace
  • Standalone: /opt/hivepro/config.yaml

Example Configuration

hivepro:
  server:
    port: 8080
    tls:
      enabled: true
      cert: /path/to/cert.pem
      key: /path/to/key.pem
  
  scanning:
    defaultSeverity: medium
    maxConcurrent: 5
    timeout: 3600
  
  integrations:
    slack:
      enabled: true
      webhook: https://hooks.slack.com/...
    jira:
      enabled: false
